# Mobile App — План развития

## Текущее состояние

### Реализовано
- **Авторизация** — OTP по телефону, выбор города, геолокация
- **Лента (Feed)** — бесконечный скролл, фильтры по городу/возрасту, карточки
- **Детали товара/услуги** — виджетная вёрстка (инфо, оплата, владелец, отзывы, расписание, контакты, команда, локация)
- **Избранное** — лайки, батч-проверка статуса, экран избранного
- **Профиль** — редактирование имени, загрузка аватара
- **Медиа** — карусель, видеоплеер
- **Тёмная тема**

### Заглушки (stub)
- Каталог — экран есть, реализации нет
- Чаты — экран есть, реализации нет

---

## Фаза 1 — Каталог и поиск

Цель: дать пользователю возможность находить услуги не только через ленту.

### 1.1 Каталог категорий
- Экран со списком категорий (`GET /categories`)
- Иконки категорий, вложенность (parent → children)
- Переход в категорию → список товаров/услуг (`GET /categories/:id/items`)
- Фильтры по категории (`GET /categories/:id/filters`): цена, рейтинг, атрибуты, расстояние, расписание

### 1.2 Поиск
- Экран поиска с текстовым вводом (`GET /search`)
- Фильтрация по категории, типу, цене
- Cursor-пагинация (бесконечный скролл)
- История поисковых запросов (локально)

**API-эндпоинты:** `GET /categories`, `GET /categories/:id/items`, `GET /categories/:id/filters`, `GET /search`

---

## Фаза 2 — Отзывы

Цель: пользователи могут оставлять и читать отзывы.

### 2.1 Просмотр отзывов
- Список отзывов на экране детали товара (`GET /reviews?targetType=item&targetId=...`)
- Отзывы организации (`GET /reviews/organization/:orgId`)
- Рейтинг, текст, дата, автор

### 2.2 Создание и управление отзывами
- Форма создания отзыва (`POST /reviews`) — рейтинг (звёзды) + текст
- Мои отзывы (`GET /reviews/my`)
- Редактирование (`PATCH /reviews/:id`) и удаление (`DELETE /reviews/:id`)
- Ответ на отзыв (`POST /reviews/:id/reply`) — для владельцев организаций
- Диспут (`POST /reviews/:id/dispute`)

**API-эндпоинты:** `GET /reviews`, `GET /reviews/my`, `POST /reviews`, `PATCH /reviews/:id`, `DELETE /reviews/:id`, `POST /reviews/:id/reply`, `POST /reviews/:id/dispute`

---

## Фаза 3 — Организации (бизнес-кабинет)

Цель: владельцы бизнеса могут управлять своей организацией из мобильного приложения.

### 3.1 Создание и управление организацией
- Создание организации (`POST /organizations`)
- Присоединение по токену (`POST /organizations/claim`)
- Просмотр и редактирование профиля организации (`GET/PATCH /organizations/:id`)
- Отправка на модерацию (`POST /organizations/:id/submit-for-moderation`)
- Снятие с публикации (`POST /organizations/:id/unpublish`)

### 3.2 Управление товарами/услугами
- Список товаров организации (`GET /organizations/:orgId/items`)
- Создание товара/услуги (`POST /organizations/:orgId/items`)
- Редактирование черновика (`PATCH /organizations/:orgId/items/:itemId`)
- Отправка на модерацию (`POST /organizations/:orgId/items/:itemId/submit-for-moderation`)
- Удаление черновика (`DELETE /organizations/:orgId/items/:itemId`)

### 3.3 Управление сотрудниками
- Список сотрудников (`GET /organizations/:id/employees`)
- Приглашение (`POST /organizations/:id/employees`)
- Изменение роли (`PATCH /organizations/:id/employees/:userId`)
- Удаление (`DELETE /organizations/:id/employees/:userId`)
- Передача владения (`POST /organizations/:id/transfer-ownership`)

### 3.4 Роли организации
- Просмотр ролей (`GET /organizations/:id/roles`)
- Создание / редактирование / удаление ролей

**API-эндпоинты:** все `/organizations/...` эндпоинты

## Фаза 4 — Взаимодействия и аналитика

Цель: трекинг поведения пользователя для улучшения рекомендаций.

### 4.1 Трекинг взаимодействий
- Отправка просмотров (`POST /interactions/views`) — при скролле ленты
- Клик по карточке (`POST /interactions/click`)
- Показ контактов (`POST /interactions/show-contacts`)
- Клик по контакту (`POST /interactions/contact-click`)

Эти события уже поддерживаются API, но не отправляются из мобильного приложения. Интеграция повысит качество рекомендаций в ленте.

**API-эндпоинты:** `POST /interactions/views`, `POST /interactions/click`, `POST /interactions/show-contacts`, `POST /interactions/contact-click`

---

## Фаза 5 — Чаты / Уведомления

Цель: коммуникация между пользователями и владельцами бизнеса.

### 5.1 Push-уведомления
- Интеграция `expo-notifications`
- Регистрация push-токена на сервере (требуется новый эндпоинт на API)
- Уведомления о новых отзывах, ответах, статусах модерации

### 5.2 Чаты (требует нового API)
- Экран списка диалогов
- Экран чата (сообщения, медиа)
- Real-time через WebSocket или SSE
- Уведомления о новых сообщениях

> **Примечание:** API для чатов и push-уведомлений ещё не реализован в main-api. Эта фаза требует параллельной работы над бэкендом.

## Фаза 6 — UX-улучшения

### 6.1 Управление сессиями
- Просмотр активных сессий (`GET /me/sessions`)
- Удаление сессии (`DELETE /me/sessions/:sessionId`)
- Завершение всех других сессий (`DELETE /me/sessions`)

### 6.2 Смена города
- Возможность сменить город из профиля или шапки
- Обновление `cityId` через `PATCH /me/profile`
- Инвалидация ленты и каталога при смене города

### 6.3 Права доступа
- Отображение доступных действий на основе `GET /me/permissions`
- Скрытие/показ UI-элементов по правам (например, кнопка "управление организацией")

### 6.4 Офлайн-режим
- Кэширование ленты и деталей для оффлайн-просмотра
- Очередь действий (лайки, отзывы) при отсутствии сети
- Синхронизация при восстановлении соединения
